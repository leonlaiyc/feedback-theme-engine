# Python Project Quality

- Prefer simple, readable, testable Python.
- Keep functions small and behavior explicit.
- Add type hints where they make code easier to understand.
- Avoid unnecessary abstractions and heavy dependencies.
- Write or update tests when behavior changes.
- Run `python -m compileall src` and `pytest` after code changes when available.
- Keep code aligned with the portfolio scope and current project phase.
