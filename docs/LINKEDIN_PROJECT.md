## Project Description

Public-facing English:

Built a portfolio-grade analytics engineering project that turns unstructured customer reviews into evidence-backed theme insights. The workflow validates local review data, creates sentence embeddings, discovers candidate themes with UMAP + HDBSCAN, explains clusters with representative examples and TF-IDF keywords, quantifies exploratory prevalence and rating-association signals, and drafts insight summaries with a mock-default LLM layer. The project emphasizes reproducibility, data safety, statistical caution, and clear separation between semantic discovery, exploratory measurement, and LLM-assisted communication.

## LinkedIn Post Draft

Public-facing English:

I finished packaging my Customer Feedback Theme Engine portfolio project.

The project turns unstructured customer reviews into evidence-backed theme insights through a reproducible Python workflow:

- local data validation and conservative text cleaning
- sentence embeddings for semantic representation
- UMAP + HDBSCAN for candidate theme discovery
- representative reviews and TF-IDF keywords for explainability
- exploratory prevalence, rating association, effect size, confidence intervals, and FDR correction
- mock-default LLM-assisted labeling and insight drafting

The main design choice was to keep discovery, measurement, and communication separate. Embeddings and clustering identify candidate themes, statistics help prioritize them cautiously, and the LLM layer only summarizes supplied evidence. The repository does not redistribute raw data, does not require paid APIs by default, and does not claim production readiness or real company conclusions.

中文翻译：

我完成了 Customer Feedback Theme Engine 作品集项目的包装整理。

这个项目通过可复现的 Python 流程，把非结构化客户评论转成有证据支撑的主题洞察：

- 本地数据验证和保守文本清洗
- 用句向量进行语义表示
- 用 UMAP + HDBSCAN 发现候选主题
- 用代表性评论和 TF-IDF 关键词提供可解释性
- 计算探索性的主题占比、评分关联、效果量、置信区间和 FDR 校正
- 使用默认模拟 LLM 辅助主题命名和洞察草稿生成

主要设计选择是把发现、衡量和表达分开。嵌入和聚类负责识别候选主题，统计方法帮助谨慎排序，LLM 层只总结已提供的证据。这个仓库不重新发布原始数据，默认不需要付费 API，也不声称生产就绪或代表真实公司的业务结论。
